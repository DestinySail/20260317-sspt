# Step 9 落地页激活验收清单

## 后台版本管理

- [ ] 打开后台赛事编辑页时，若该赛事已有保存的落地页版本，即使没有任何版本处于激活状态，也能看到“落地页版本”区块。
- [ ] 版本列表展示每条落地页的 version、风格提示词、生成时间和“当前激活 / 未激活”状态。
- [ ] 点击任意版本行的“查看”时，打开的是该版本自己的落地页内容，而不是当前激活版本。
- [ ] 点击未激活版本的“激活”后，该版本成为当前激活版本，其他版本不再激活。
- [ ] 激活后刷新后台编辑页、后台赛事列表、首页赛事列表和 `/events/[slug]/landing`，看到的都是同一个当前激活版本。

## 前台访问

- [ ] 首页赛事列表中，已有激活落地页的赛事卡片整体点击后进入 `/events/[slug]/landing`。
- [ ] 没有激活落地页的赛事卡片仍进入 `/events/[slug]` 赛事详情页。
- [ ] `/events/[slug]/landing` 在没有激活版本时显示“暂无激活的落地页”，不报错。

## AI 生成内容

- [ ] AI 返回以 Markdown 代码块包裹的 HTML（例如以 ```html 开始）时，保存后的落地页源码不包含开头或结尾代码围栏。
- [ ] 打开 `/events/[slug]/landing` 时，页面左上角不出现 ```html、``` 或其他生成过程标记。
- [ ] 生成提示词要求 AI 按 Web Design Engineer 工作流建立视觉方向、自检视觉层级，并输出完整可渲染 HTML。
- [ ] 生成页代码区只在内部滚动，不撑出浏览器页面级上下滚动条。
- [ ] 生成页代码区使用不透明深色底，只有代码区顶部有渐隐效果，主体代码保持清晰可读。
- [ ] 代码生成时按前端打字机缓冲逐帧展示，不随网络 chunk 大段跳动。
